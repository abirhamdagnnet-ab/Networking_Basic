The OSI Model
7 LAYERS — MEMORIZE THESE

The OSI model describes how data travels from one device to another. Every networking interview will ask about this. Use the mnemonic: "Please Do Not Throw Sausage Pizza Away" (Physical → Data Link → Network → Transport → Session → Presentation → Application)

# LAYER WHAT IT DOES EXAMPLES
7 Application Interface for the user and software. Where you interact with the network. HTTP, FTP, DNS, SMTP
6 Presentation Translates, encrypts, and compresses data into readable format. SSL/TLS, JPEG, ASCII
5 Session Manages sessions (open, maintain, close connections between apps). NetBIOS, RPC, SQL
4 Transport Ensures reliable delivery. Breaks data into segments. TCP vs UDP. TCP, UDP
3 Network Routing — finds the best path between networks using IP addresses. IP, ICMP, Routers
2 Data Link Transfers data between directly connected nodes using MAC addresses. Ethernet, Switches, MAC
1 Physical Actual hardware — cables, signals, bits. The raw electrical transmission. Cables, Hubs, Wi-Fi signals
💡 Bank Context IMPORTANT

In a bank, Layer 3 (routing between branches via MPLS), Layer 4 (TCP ensures transactions are complete), and Layer 6 (TLS encrypts customer data) are especially critical. When an ATM fails, engineers check all layers from Physical (cable ok?) up to Application (server responding?).

Key Protocols
KNOW THESE BY HEART

Protocols are rules that define how devices communicate. In banking, these are used every second across ATMs, branch connections, and online banking systems.

TCP
Connection-based
Reliable, ordered delivery. Used when data must arrive complete — banking transactions, file transfers.
UDP
Connectionless
Fast, no guarantee. Used for video streaming, VoIP calls, DNS queries.
IP
Layer 3
Internet Protocol — assigns addresses and routes packets between networks.
DNS
Port 53
Translates domain names to IPs. Like a phone book for the internet.
DHCP
Port 67/68
Automatically assigns IP addresses to devices on a network.
HTTP/S
Port 80 / 443
Web traffic. HTTPS = encrypted (TLS). Banks only use HTTPS for online banking.
SSH
Port 22
Secure remote login to servers and network devices. Replaces old Telnet.
FTP/SFTP
Port 21 / 22
File transfer. SFTP is the secure version used in banks.
SMTP
Port 25 / 587
Sending email. Used by mail servers to relay messages.
SNMP
Port 161
Network monitoring — collects data from routers, switches for management tools.
ICMP
No port
Used by ping and traceroute to test connectivity and diagnose network issues.
VoIP
RTP / SIP
Voice over IP — how bank phone systems (IP PBX) work internally.
Network Devices
HARDWARE YOU WILL SEE IN THE BANK
🔀 Switch LAYER 2

Connects devices within the same network (LAN). Uses MAC addresses to forward frames only to the correct port. Banks have many switches — one per floor or department. Supports VLANs to separate traffic (e.g., staff vs ATM network).

🌐 Router LAYER 3

Connects different networks together. Uses IP addresses and routing tables to decide where to send packets. The bank's router connects the branch to the internet or to other branches via MPLS/WAN links.

🔥 Firewall SECURITY

Controls what traffic enters and leaves the network based on rules (ACLs). Banks have enterprise firewalls (Cisco, Palo Alto, Fortinet). They create a DMZ (Demilitarized Zone) to isolate public-facing services from the internal network.

📡 Access Point (WAP) WIRELESS

Provides Wi-Fi connectivity. In banks, staff Wi-Fi is separate from guest Wi-Fi, and both are isolated from the core banking network using VLANs.

🔄 Load Balancer ADVANCED

Distributes traffic across multiple servers so no single server gets overwhelmed. Critical in banking for online banking portals and ATM switch servers.

🔒 VPN Concentrator SECURITY

Manages VPN connections for remote users. Bank employees working remotely connect through VPN to access internal systems securely.

💡 Key Difference to Remember

Hub → broadcasts to ALL ports (old, dumb, not used anymore)
Switch → sends only to the correct port (smart, uses MAC table)
Router → connects different networks, uses IP addresses

Network Security
CRITICAL IN BANKING
🛡️ CIA Triad MUST KNOW

The foundation of all security:

Confidentiality — only authorized people can see data (encryption, access control)
Integrity — data is not tampered with (hashing, digital signatures)
Availability — systems are always accessible when needed (redundancy, backups)

🔐 VPN Types

Site-to-Site VPN — connects two branch offices securely over the internet. All branch traffic is encrypted.

Remote Access VPN — individual employees connect from home. They use a VPN client (like Cisco AnyConnect).

SSL VPN — browser-based VPN, no client software needed.

🚨 IDS vs IPS

IDS (Intrusion Detection System) — monitors and alerts on suspicious activity. Does NOT block anything. Like a security camera.

IPS (Intrusion Prevention System) — monitors AND actively blocks threats in real time. Like a security guard.

🌐 Network Segmentation

Banks divide their network into separate zones so a breach in one area cannot spread. Common segments:

DMZ — public-facing servers (web, email)
Staff VLAN — internal employee network
ATM Network — completely isolated for ATM transactions
Management VLAN — for IT team to manage devices

🔒 Encryption Basics

SSL/TLS — encrypts data in transit (HTTPS websites, online banking)
AES — standard encryption algorithm for data at rest
RSA — public key encryption used in certificates and authentication
Hashing (SHA-256) — converts data to a fixed string; verifies integrity without revealing content

⚠️ Common Threats in Banking

Phishing — fake emails tricking staff into giving credentials
Man-in-the-Middle — attacker intercepts network traffic
DDoS — flooding servers to cause downtime
Insider Threats — employees misusing access to sensitive data
ATM Skimming — physical + network attack on ATM systems

Essential Commands
TOOLS YOU WILL USE EVERY DAY
🖥️ Windows Commands MOST COMMON
C:\>ipconfig— show your IP, subnet, gateway
C:\>ipconfig /all— show detailed info including MAC address
C:\>ping 8.8.8.8— test connectivity to Google DNS
C:\>tracert 8.8.8.8— trace the path packets take
C:\>nslookup google.com— query DNS for an IP address
C:\>netstat -an— show all active connections and ports
C:\>arp -a— show ARP table (IP to MAC mappings)
C:\>route print— display routing table
🖧 Cisco IOS Commands SWITCHES & ROUTERS
Switch#show ip interface brief— quick view of all interfaces
Switch#show vlan brief— list all VLANs and assigned ports
Switch#show mac address-table— view MAC address table
Router#show ip route— display the routing table
Router#show running-config— see current active configuration
Router#show interfaces— detailed interface status and stats
Router#ping 192.168.1.1— test reachability from the router
Router#show cdp neighbors— see connected Cisco devices
🐧 Linux Commands SERVERS
$ifconfig— show network interfaces (older Linux)
$ip addr show— modern way to show IP addresses
$ping -c 4 8.8.8.8— ping 4 times then stop
$traceroute 8.8.8.8— trace route (Linux version)
$netstat -tuln— show listening ports
$ss -tulnp— modern socket stats
